How It Works

1

Secure Login & Authentication

User credentials are encrypted and validated. A secure **JWT** is generated and sent to the client, enabling access to protected API routes for personalized learning sessions.

2

Client-Server API Interaction

The frontend client sends user input to the **Node.js REST API**. The request, authorized by the JWT, is processed, and conversation history is retrieved from the database.

3

Real-Time AI Response

The backend securely communicates with the **OpenAI/Gemini API**, sending the user's message and context. The AI generates a tailored response, which is then sent back to the client.

4

Learning Feedback Loop

The conversation turn is persisted in the database, updating the user's progress. The frontend state is updated, displaying the new message and completing the learning loop.

Frequently Asked Questions

What is the complete tech stack for this project?

add remove

The project utilizes a modern full-stack JavaScript architecture:
- **Frontend:** React, Tailwind CSS, State Management (e.g., Redux Toolkit)
- **Backend:** Node.js, Express.js
- **Database:** MongoDB with Mongoose ODM
- **Authentication:** Custom JWT implementation with bcrypt for password hashing
- **AI Integration:** OpenAI/Gemini API
- **Deployment:** Docker, AWS/Vercel

What was the biggest technical challenge you overcame?

add remove

The most significant challenge was designing and implementing the custom JWT authentication from scratch. This involved creating secure token generation, implementing middleware for route protection, handling token expiration and refresh logic, and ensuring that user data was securely associated with each session. It required a deep understanding of stateless authentication flows, security best practices, and asynchronous error handling in a Node.js environment.

How is user data and session state managed?

add remove

User data, including credentials (hashed) and conversation history, is stored in a MongoDB database. Session state is managed on the client-side using the JWT stored securely (e.g., in an HttpOnly cookie or local storage). The server remains stateless; every authenticated request includes the JWT, which the backend decodes and verifies to identify the user and authorize access to their specific data, ensuring a secure and scalable architecture.
